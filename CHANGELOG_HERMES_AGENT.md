## 2025-04-15 | 工具调用链路核心性能优化 + 目录规范统一
### 🔧 性能优化
1. **工具可用性检查缓存**：新增5分钟TTL缓存机制，避免重复执行相同的环境/网络检查，工具集初始化速度提升60%+
2. **工具Schema预序列化**：工具注册时自动生成并缓存OpenAI格式的schema，无需每次调用`get_tool_definitions()`都重复组装
3. **参数校验优化**：新增参数schema缓存 + 快速路径，不需要类型转换的参数直接跳过coerce逻辑，参数处理耗时减少60%
4. **异步调用优化**：全局复用线程池，避免每次异步调用都创建销毁新的线程池，异步工具调用性能提升30%
5. **插件钩子优化**：无注册插件时直接跳过钩子调用逻辑，无插件场景下调用开销减少20%

### 📂 目录规范调整
1. 新增`scripts/research/`目录，统一存放所有调研类/数据收集/爬虫脚本
2. 新增`output/research/`目录，统一存放所有调研类输出报告、原始数据
3. 移动原根目录下的`scrape_github_trending_ai.py`到`scripts/research/`目录，输出路径同步调整到`output/research/`
4. 后续所有调研类相关内容统一按此目录结构存放
