# Hermes Agent 技术架构分析

## 核心定位
Hermes Agent 是一款完全本地化运行、支持自主进化的通用AI助手，核心特性是所有用户数据完全存储在用户本地设备，不依赖云端，可随用户使用习惯持续迭代适配。

---

## 技术栈
| 模块 | 技术选型 | 说明 |
|------|---------|------|
| 开发语言 | Python 3.10+ | 全栈Python实现，依赖轻量，跨平台兼容性好 |
| 网络请求 | httpx | 异步+同步兼容的HTTP客户端 |
| 本地存储 | SQLite + FTS5 | 原生SQLite支持全文检索，无需额外数据库服务 |
| CLI交互 | Rich + Prompt Toolkit | Rich做界面渲染，Prompt Toolkit做交互式输入补全 |
| 浏览器自动化 | Browserbase | 支持无界面浏览器操作、截图分析 |
| 扩展协议 | MCP (Model Context Protocol) | 原生支持对接任意符合MCP协议的第三方工具/服务 |

## 代码架构分层
### 1. 核心入口层
```
├── run_agent.py          # 核心会话循环，AIAgent类实现，所有交互入口
├── cli.py                # 命令行入口，`hermes xxx`命令调度中心
├── model_tools.py        # 工具调度引擎，解析工具调用、执行、返回结果
├── toolsets.py           # 内置工具集定义，控制对外开放的工具范围
└── hermes_state.py       # SQLite存储实现，会话历史、记忆、任务数据持久化
```

### 2. 核心引擎层 (`agent/` 目录)
```
├── prompt_builder.py     # 提示词组装：自动拼接系统提示、历史会话、记忆、技能
├── context_compressor.py # 上下文自动压缩，超长会话自动精简保留关键信息
├── skill_commands.py     # 技能调度逻辑，处理技能的加载、执行、更新
├── model_metadata.py     # 模型元数据管理，不同模型的上下文长度、token估算逻辑
└── display.py            # 界面渲染，加载动画、工具结果展示样式
```

### 3. CLI实现层 (`hermes_cli/` 目录)
```
├── commands.py           # 所有 slash 命令（/model /skills /help等）定义
├── setup.py              # 首次启动交互式设置向导
├── skin_engine.py        # CLI主题引擎，支持自定义界面样式
├── model_switch.py       # 模型切换逻辑，支持动态切换不同大模型提供商
└── skills_hub.py         # 技能市场实现，支持浏览、安装社区技能
```

### 4. 工具能力层 (`tools/` 目录)
```
├── registry.py           # 工具注册中心，所有工具需在此注册后可调用
├── terminal_tool.py      # 终端命令执行工具，支持前台/后台/沙箱运行
├── file_tools.py         # 文件读写、搜索、修改工具
├── browser_tool.py       # 浏览器自动化工具，支持网页访问、交互、截图分析
├── mcp_tool.py           # MCP协议客户端，支持对接任意第三方MCP服务
├── code_execution_tool.py# Python代码沙箱执行工具
└── environments/         # 多环境执行后端，支持本地/Docker/SSH/Modal/云沙箱等
```

### 5. 多平台接入层 (`gateway/` 目录)
```
├── run.py                # 网关主循环，对接第三方平台的消息收发
└── platforms/            # 各平台适配器：飞书、Telegram、Discord、Slack、微信等，每个平台独立文件实现
```

### 6. 扩展能力层
```
├── cron/                 # 定时任务调度实现
├── acp_adapter/          # IDE适配层，支持对接VS Code、Zed、JetBrains等IDE
└── skills/               # 本地技能库，存储所有可复用工作流程
```

## 核心运行流程
完全同步的极简执行逻辑，易于理解和二次开发：
1. 接收用户消息
2. 自动组装上下文（历史会话 + 持久记忆 + 匹配到的技能）
3. 调用大模型生成响应
4. 如果大模型返回工具调用指令，执行工具获取结果
5. 将工具结果返回给大模型继续处理
6. 重复步骤4-5直到大模型输出最终回复
7. 将最终回复返回给用户

## 架构优势
1. **高度解耦**：各模块职责清晰，耦合度极低，扩展成本小
   - 新增工具：仅需在`tools/`目录新增文件，注册到registry即可，无需修改核心代码
   - 新增平台：仅需在`gateway/platforms/`新增适配文件，实现消息收发逻辑即可
   - 新增技能：直接将技能文件放到`~/.hermes/skills/`目录即可自动识别
   - 新增模型：仅需在`hermes_cli/models.py`添加对应模型元数据即可支持
2. **稳定性高**：3000+ pytest测试用例覆盖核心功能，错误处理、边界校验、降级逻辑完善
3. **本地化优先**：所有数据默认存储在用户本地，隐私性强，支持完全离线运行
4. **可进化性**：支持技能自动迭代、记忆自动更新，可随用户使用习惯持续适配
